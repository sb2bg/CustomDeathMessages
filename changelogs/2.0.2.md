# CustomDeathMessages 2.0.2

Changes since `d0246a989ed851d9f591e301af43e61946eea5e3`

## Major Features

- Added Discord integration support for both `EssentialsDiscord` and `DiscordSRV`.
- Added proper `DiscordSRV` death message forwarding with pre-process and post-process hook support.
- Added Discord fallback logic so `EssentialsDiscord` is preferred when its service is available, with automatic fallback to `DiscordSRV`.
- Added a demo screenshot to the project README.

## Compatibility and Stability

- Fixed modern Paper compatibility by replacing brittle CraftBukkit package parsing with Bukkit version parsing.
- Removed flaky held-item version gating and switched to a safer main-hand lookup with legacy fallback.
- Fixed delayed-death propagation issues by storing propagated death context longer and expiring it on access instead of with async timers.
- Removed async mutation of propagation state, avoiding unsafe access patterns around delayed deaths.
- Fixed the head-drop config mismatch so the documented `drop-head-percentage` key is actually honored.
- Fixed Java 8 compatibility in `DeathCause` by removing Java 9+ `Map.ofEntries` usage.
- Rebuilt cleanly against the current shaded dependency set to avoid the bStats constructor mismatch seen in inconsistent local builds.

## Death Cause Coverage

- Added support for additional damage and entity mappings including `WORLD_BORDER`, `CAMPFIRE`, `KILL`, `POISON`, `DRYOUT`, `MELTING`, `ENTITY_SWEEP_ATTACK`, `BOGGED`, and `BREEZE`.
- Added alias handling for names like `TNT`, `TNT_MINECART`, `MINECART_TNT`, `FIREWORK_ROCKET`, `END_CRYSTAL`, and `LIGHTNING_BOLT`.
- Reworked death-cause lookup to use cached reverse maps instead of repeated enum scans.
- Reworked path-to-cause lookup and grouped-path reverse lookup for faster command completion and list resolution.

## Config and Data Handling

- Replaced the old document-loading flow with a generalized `BaseDocumentManager` that loads, saves, reloads, auto-updates, and disables safely on fatal config errors.
- Migrated config loading to use `config.yml` directly rather than the older `document.yml` flow.
- Added config updater and versioning support and bumped `config-version` to `3`.
- Improved reload handling so reload failures return clear command feedback and successful reloads clear cached cooldown and propagation state.
- Consolidated config persistence through a shared save path instead of repeated ad hoc document saves.
- Updated metrics keys to match current config keys.

## Commands and UX

- Added `/deathmessages` as a registered alias alongside the existing command aliases.
- Improved `/cdm reload` feedback so failures explicitly tell the user to check console output.
- Kept debug commands gated behind `developer-mode`.

## Default Messages and Config Defaults

- Refreshed a large portion of the default death-message set with cleaner wording and broader coverage.
- Added default `breeze-messages`.
- Added additional defaults for many existing categories including firework, void, spider, drowned, pillager, ravager, bee, stray, ghast, guardian, fox, and more.
- Changed `enable-custom-name-entity-messages` default to `false`.

## Build and Dependency Updates

- Updated Maven compiler and shade plugin configuration.
- Added `finalName` so the packaged jar is consistently named `CustomDeathMessages.jar`.
- Updated `spigot-api`, `bstats-bukkit`, `nbteditor`, and `boosted-yaml`.
- Added Maven repositories required for the Discord integrations and newer dependency set.
- Added provided dependencies for `EssentialsXDiscord` and `DiscordSRV`.
- Released version string updated to `2.0.2`.

## Documentation

- Rewrote the README to match the actual shipped feature set instead of the older plugin state.
- Documented the real command surface, permissions, config keys, placeholders, Discord integrations, and fallback behavior.
- Added the demo screenshot to the top of the README.
