# CustomDeathMessages 2.0.2 User Changelog

## Highlights

- Minecraft 1.21+ Support!
- Added Discord support for both `EssentialsDiscord` and `DiscordSRV`.
- Added support for newer Minecraft damage and mob types, including `Breeze` and `Bogged`.
- Improved compatibility with modern Paper versions.
- Improved delayed death handling so custom death messages are less likely to lose killer/cause data when other plugins delay death processing.

## Commands and Configuration

- Added `/deathmessages` as a command alias.
- Improved `/cdm reload` feedback when reload fails.
- Fixed the head drop chance config so `drop-head-percentage` works as documented.
- Updated and expanded the default config and death-message lists.

## Defaults and Documentation

- Refreshed many bundled death messages with cleaner wording and broader coverage.
- Added default messages for `Breeze`.
- Changed `enable-custom-name-entity-messages` to default to `false`.
- Updated the README to better match the plugin's current features, commands, placeholders, and integrations.
- Added a demo screenshot to the project documentation.
